# 体裁本质差异分析

> 创建日期: 2026-07-19 | 本文档是架构设计的理论基础
>
> 每个体裁的叙事DNA完全不同。本文档分析每个体裁在"写什么、怎么写、怎么评判"三个维度上的本质差异，作为后续工具设计、前端设计、知识图谱扩展的依据。

---

## 一、分析框架：六个维度

对每个体裁，从以下六个维度做本质分析：

| 维度 | 含义 | 决定什么 |
|------|------|---------|
| **叙事单元** | 最小的不可分割的内容单元是什么？ | 编辑器设计、存储结构 |
| **表达媒介** | 通过什么载体传达信息？ | 输出格式、工具设计 |
| **信息密度分布** | 信息在哪些维度上需要被精确控制？ | 知识图谱 schema、数据模型 |
| **消费模式** | 读者/观众/玩家如何消费内容？ | 节奏控制、评审标准 |
| **评判标准** | 什么是"好"？ | 评审团设计、质量门禁 |
| **创作流程** | 从构思到完成的标准工作流？ | 工作流模板、Agent 行为 |

---

## 二、各体裁深度分析

### 2.1 小说（novel）

**叙事单元**: 章 → 节 → 段落 → 句子

**表达媒介**: 纯文字。作者通过文字在读者脑中构建画面。可以写内心独白、环境描写、叙述者评论、意识流。文字是唯一的载体。

**信息密度分布**:
- 高密度：角色内心活动、叙述者视角、环境氛围
- 中密度：对白、动作描写
- 低密度：精确的视觉细节（读者自己想象）
- 小说不需要写：景别、运镜、布景清单、灯光方案

**消费模式**: 线性、沉浸式、长时间（一次读几千到几万字）。读者自己控制节奏。

**评判标准**: 文笔优美度、人物塑造深度、情节合理性、主题深度、情感共鸣

**创作流程**: 大纲 → 逐章写作 → 评审 → 修改 → 定稿

**与现有系统的关系**: 当前系统完全按小说设计。`write_chapter`/`delegate_writing`/`chapter_store` 等工具和数据结构都是小说原生。

---

### 2.2 短剧剧本（short_drama）——用户重点关注的例子

**叙事单元**: 集 → 场景 → 镜头（分镜）

**表达媒介**: 视觉画面 + 对白 + 动作 + 镜头语言。**没有叙述者**。所有信息必须通过画面和对白传达。内心活动必须外化为动作或对白。

**信息密度分布**:
- 高密度（短剧特有）：景别、运镜、布景、灯光、服装、演员走位、音效、BGM、画面构图、时长
- 中密度：对白、动作描述
- 低密度：内心活动（不能写！必须外化）
- 短剧特有的信息维度：
  - 景别：特写/近景/中景/全景/远景 — 决定观众看到什么
  - 运镜：推/拉/摇/移/跟/升/降 — 决定画面如何运动
  - 布景：场景的空间布局、道具清单、氛围色调
  - 钩子：每集结尾的悬念设计
  - 爽点标记：打脸/反转/逆袭/身份揭晓的时刻
  - AI视频提示词：最终输出给AI视频生成工具的完整描述

**消费模式**: 竖屏、1-2分钟/集、碎片化、视觉冲击优先。观众在刷抖音，3秒内抓不住就划走了。

**评判标准**: 爽点密度（每集至少1个）、反转节奏（30秒一个小高潮）、钩子效果（每集结尾必须有）、画面冲击力、对白口语化程度

**创作流程**: 故事大纲 → 分集大纲 → 逐集剧本（场景+分镜+对白） → AI视频提示词

**与小说的本质差异**:
```
小说写：                         短剧剧本写：
"他愤怒地摔门而出，               【场景3】客厅 - 日
在走廊里走了几步，                【中景】门被猛地推开，撞到墙上反弹
又停下来，靠在墙上。"              【跟拍】角色快步走过走廊
                                  【灯光】走廊顶灯闪烁，投下不安的阴影
                                  【特写】紧握的拳头，指节发白
                                  【对白】（无对白，纯动作表达愤怒）
                                  【时长】8秒
                                  【钩子】走廊尽头，一个黑影闪过
```

**关键洞察**: 短剧不是"写得更短的小说"。短剧是**完全不同的媒介**——它用画面思考，而不是用文字思考。一个短剧作者需要同时兼任导演、摄影师、灯光师、美术指导。

### 2.3 剧本杀（murder_mystery）

**叙事单元**: 角色剧本 → 线索卡 → 时间线 → 真相复盘

**表达媒介**: 碎片化文本。每个角色拿到的是不完整的信息。真相分散在多个文档中。玩家通过拼图还原真相。

**信息密度分布**:
- 高密度（剧本杀特有）：时间线（精确到分钟）、线索物理属性、不在场证明、动机链、人物关系网
- 中密度：角色背景、对话
- 低密度：环境描写（DM可以即兴发挥）
- 剧本杀特有的信息维度：
  - 多视角时间线：每个角色有独立的时间线，包含空白、谎言、错位
  - 线索卡：编号、类型、内容、来源、层级、指向性
  - 不在场证明：每个嫌疑人在案发时间的行踪
  - 动机链：深层动机 → 触发事件 → 作案动机
  - 谎言表：每个角色说了什么谎，为什么说谎
  - 红鲱鱼：故意误导的线索

**消费模式**: 多人互动、阅读+推理+讨论。玩家同时阅读各自剧本，然后通过对话交换信息。DM控制节奏。

**评判标准**: 诡计逻辑自洽性（不能有bug）、线索可推理度（不能太难也不能太简单）、角色平衡性（每个角色都有参与感）、复盘完整性（真相必须能还原）、游戏时长合理性

**创作流程**: 核心诡计设计 → 角色设计 → 时间线设计 → 线索卡设计 → 各角色剧本写作 → 真相复盘 → 测试（模拟玩家推理）

**与小说的本质差异**:
```
小说写：                         剧本杀写：
"凶手在17:00进入房间，            【角色A的时间线视角】
杀死了受害者，然后离开。"          16:50 到达宴会厅，看到B和C在交谈
                                  17:00 去洗手间（此段空白——A其实在作案）
                                  17:15 返回宴会厅，听到D说"E怎么还没来"
                                  17:30 发现尸体
                                  
                                  【线索卡#7】破碎的表
                                  类型：物证 | 来源：洗手间垃圾桶
                                  内容：一块停止在17:03的手表，表盘碎裂
                                  层级：深入 | 指向：角色A的时间线谎言
```

**关键洞察**: 剧本杀不是"多视角的小说"。它是**信息不对称的推理游戏**。作者不是在写一个故事，而是在设计一个谜题。每个角色剧本是谜题的一部分，线索卡是谜题的碎片，真相复盘是谜题的答案。

### 2.4 游戏文案（game_narrative）

**叙事单元**: 任务 → 步骤 → 对话节点 → 条件分支

**表达媒介**: 对话 + 物品描述 + 场景旁白 + UI文本 + 语音文本。信息分散在多个载体中。玩家主动触发，而非被动接收。

**信息密度分布**:
- 高密度（游戏特有）：对话树（分支+条件）、物品描述、UI文本、语音文本、任务引导
- 中密度：角色对话、场景旁白
- 低密度：长篇叙述（玩家会跳过）
- 游戏特有的信息维度：
  - 对话树：NPC对话 → 玩家选项 → 条件判断 → 分支结果
  - 物品描述：装备/道具/材料的文字说明，透露世界观
  - 碎片化叙事：世界观通过物品、场景、NPC日常对话拼接
  - 任务链：前置条件 → 任务步骤 → 奖励 → 后续影响
  - 角色语音：每个角色的语音文本（用于配音）
  - 多语言：需要支持本地化

**消费模式**: 交互式、玩家驱动、非线性。玩家可以选择何时触发什么对话，可以跳过，可以重复。文本长度受限（对话框有字数限制）。

**评判标准**: 对话自然度、角色性格一致性、世界观碎片自洽性、任务引导清晰度、二次元风格贴合度、配音适配性

**创作流程**: 世界观设定 → 角色设定 → 任务树设计 → 对话树设计 → 碎片化叙事（物品/场景） → 配音文本 → 本地化

**与小说的本质差异**:
```
小说写：                         游戏文案写：
"艾莉丝看着远方，回忆起             【NPC: 艾莉丝】
故乡的樱花。"                     （日常对话，好感度≥5触发）
                                  "这樱花……让我想起故乡。
                                  那里的樱花比这里更粉一些。
                                  ……算了，不提了。"
                                  
                                  【物品: 樱花发饰】
                                  稀有度：★★★
                                  "艾莉丝随身携带的发饰，花瓣
                                  已经有些褪色。据说来自她的故乡。"
                                  
                                  【场景: 樱花树下】
                                  "一棵孤独的樱花树，花瓣飘落
                                  在空无一人的长椅上。"
```

**关键洞察**: 游戏文案不是"小说里的对话摘出来"。它是**分散在多个载体中的碎片化叙事**。同一个世界观的设定，可能出现在NPC的日常对话中、物品的描述文字中、场景的环境叙事中、UI的提示文字中。玩家通过拼接这些碎片来理解世界观。

---

### 2.5 互动小说/AVG（interactive_fiction）

**叙事单元**: 场景 → 对话 → 选择节点 → 分支

**表达媒介**: 文字 + 立绘 + 背景 + 选项。文字仍是主要载体，但被选项打断。视觉元素（立绘变化、背景切换）是叙事的一部分。

**信息密度分布**:
- 高密度（AVG特有）：选择节点、好感度值、属性值、分支条件、CG触发条件、结局条件
- 中密度：场景描述、对话、叙述
- 低密度：复杂的内心独白（被选项打断）
- AVG特有的信息维度：
  - 分支树：场景 → 选择A/B/C → 分支A1/A2/A3 → ...
  - 好感度系统：每个角色好感度值（0-100），对话选择影响数值
  - 属性系统：勇气/智力/魅力等，影响可选选项
  - CG触发：特定好感度+特定场景 → 触发CG
  - 结局条件：True End / Normal End / Bad End / Secret End 的触发条件
  - 锁定的选项：条件不满足时显示但不可选

**消费模式**: 阅读为主，关键点选择，多周目。玩家会反复玩以解锁不同结局。文本量巨大（多分支）。

**评判标准**: 分支有意义性（选A和B真的不同）、结局多样性、好感度逻辑合理性、CG触发合理性、多周目可玩性

**创作流程**: 大纲（分支树） → 角色设定 → 好感度/属性系统设计 → 场景写作 → 分支写作 → 结局设计 → CG触发设计

**与小说的本质差异**:
```
小说写：                         互动小说写：
"她选择了左边的路。"              [选择节点]
                                  → A. "走左边的路" [好感度+5]
                                    分支A：她走向左边，发现...
                                  → B. "走右边的路" 
                                    分支B：她走向右边，发现...
                                  → C. "站在原地" [条件: 智力≥5]
                                    [锁定] 智力不足，无法做出判断
```

**关键洞察**: 互动小说不是"有选项的小说"。它是**状态机驱动的叙事**。作者不是在写一个故事，而是在设计一个包含多个状态（好感度、属性、物品）和多个路径（分支）的系统。同一个场景可能因为好感度不同而有完全不同的对话。

### 2.6 广播剧/有声书（audio_drama）

**叙事单元**: 集 → 场景 → 旁白/对白 → 音效/BGM

**表达媒介**: 纯音频。没有画面。所有信息通过声音传达——对白、旁白、音效、BGM、静默。

**信息密度分布**:
- 高密度（广播剧特有）：音效标注、BGM提示、配音指导（语气/情绪/语速）、空间感（远近/左右声道）
- 中密度：对白、旁白
- 低密度：视觉描述（听众看不到）
- 广播剧特有的信息维度：
  - 音效：环境音、动作音、转场音效
  - BGM：情绪基调、转场音乐
  - 配音指导：语气、情绪、语速、音量
  - 空间感：声音的远近（脚步声渐近）、左右声道（角色在左边说话）
  - 静默：有意义的沉默（比对话更有力量）

**消费模式**: 纯听觉、线性、沉浸式。听众在做其他事时消费（开车、做家务）。不能有复杂的信息密度——听众不能"回看"。

**评判标准**: 对白自然度、声音画面感、配音适配性、音效设计、节奏（不能太密也不能太疏）

**关键差异**: 广播剧取消了"画面"这个维度，但增加了"声音空间"这个维度。

---

### 2.7 虚拟主播/虚拟人（virtual_host）

**叙事单元**: 段 → 互动 → 话题

**表达媒介**: 直播脚本 + 实时互动。即兴感很重要。需要预留互动空间。

**关键差异**: 这不是"写好的文本"，而是"半即兴的演出框架"。脚本给出骨架，虚拟主播在框架内即兴发挥。需要处理粉丝互动（弹幕/评论）。

---

### 2.8 跑团模组（ttrpg）

**叙事单元**: 场景 → 房间 → 遭遇 → 检定

**表达媒介**: DM指南 + 玩家可探索的空间。信息不对称（DM知道全部，玩家逐步探索）。

**关键差异**: 不是"写好的故事"，而是"可探索的空间+可触发的遭遇"。DM不是作者，而是裁判+讲述者。需要数值系统（DC/AC/HP等）。

---

### 2.9 漫画脚本（comic_script）

**叙事单元**: 话 → 页 → 格 → 画面

**表达媒介**: 分格画面 + 对话气泡 + 旁白框。每一格是一个静态画面。画面之间的跳跃（格与格之间）由读者脑补。

**关键差异**: 漫画脚本写的是"给画师的指令"。需要精确描述每格的画面内容、角色位置、角度、表情、对话气泡位置和顺序。

---

### 2.10 儿童故事/绘本（children_story）

**叙事单元**: 篇 → 页 → 画面 + 文字

**表达媒介**: 大画面 + 简短文字。文字是画面的补充，画面是主要的叙事载体。

**关键差异**: 词汇受年龄分级严格限制。教育目标嵌入。亲子互动提示。画面描述是给插画师的。

---

### 2.11 品牌世界观（brand_world）

**叙事单元**: 文档 → 设定 → 规则

**表达媒介**: 品牌指南文档。不是讲故事，而是定义"可以讲什么故事"的规则。

**关键差异**: 这不是创作，而是**元创作**——定义叙事边界和规则。输出是给其他创作者的约束文档。

---

## 三、差异汇总矩阵

| 体裁 | 叙事单元 | 主导媒介 | 是否有"叙述者" | 是否视觉化 | 是否有"状态" | 是否碎片化 | 是否互动 |
|------|---------|---------|--------------|-----------|-------------|-----------|---------|
| 小说 | 章节 | 文字 | ✅ 有 | ❌ 脑补 | ❌ | ❌ | ❌ |
| 短剧剧本 | 场景→镜头 | 画面+对白 | ❌ 无 | ✅ 精确 | ❌ | ❌ | ❌ |
| 剧本杀 | 角色剧本→线索 | 碎片文本 | ❌ 无 | ❌ | ❌ | ✅ 多视角 | ✅ 玩家推理 |
| 游戏文案 | 任务→对话节点 | 对话+物品+UI | ❌ 无 | ❌ | ❌ | ✅ 多载体 | ✅ 玩家触发 |
| 互动小说 | 场景→选择→分支 | 文字+选项 | ✅ 有 | ❌ 立绘 | ✅ 好感度/属性 | ❌ | ✅ 选择分支 |
| 广播剧 | 场景→对白 | 声音 | ✅ 旁白 | ❌ | ❌ | ❌ | ❌ |
| 虚拟主播 | 段→互动 | 声音+画面 | ❌ 无 | ❌ | ❌ | ❌ | ✅ 实时互动 |
| 跑团模组 | 场景→房间 | DM指南 | ❌ 无 | ❌ | ✅ 数值系统 | ✅ 探索 | ✅ 玩家决策 |
| 漫画脚本 | 页→格 | 画面 | ❌ 无 | ✅ 精确 | ❌ | ❌ | ❌ |
| 儿童故事 | 页→画面 | 画面+文字 | ✅ 有 | ✅ 精确 | ❌ | ❌ | ❌ |
| 品牌世界观 | 文档 | 规则文本 | ❌ 无 | ❌ | ❌ | ❌ | ❌ |

## 四、核心结论

### 4.1 不能共用的东西

每个体裁需要**独立设计**的：

1. **编辑器**：小说用 Markdown 编辑器，短剧用分镜编辑器（时间轴+卡片），剧本杀用多面板编辑器（角色A/角色B/线索卡同时可见），游戏文案用对话树可视化编辑器
2. **评审标准**：短剧评"爽点密度"，剧本杀评"诡计自洽性"，游戏文案评"碎片自洽性"
3. **内容单元数据结构**：小说的 `{title, content, status}` 不适用于剧本杀的 `{role_name, public_info, secret, timeline, motive}` 或短剧的 `{scene_number, shot_type, camera_move, dialogue, action, hook}`
4. **知识图谱 schema**：剧本杀需要 Clue/SuspectRole/Alibi，游戏文案需要 Quest/DialogueNode/Item

### 4.2 可以共用的东西

1. **Agent 循环引擎**（`agent_loop.py`）—— 体裁无关
2. **知识图谱基础**（Entity/Relation/TimelineEvent）—— 所有体裁都有角色/地点/事件
3. **工具注册/分发机制**（`DISPATCH` 表）—— 机制通用，工具不同
4. **幻觉检测** —— 与体裁无关
5. **上下文管理/Token预算** —— 与体裁无关
6. **LLM客户端** —— 与体裁无关

### 4.3 架构设计原则

**每个体裁是一个独立的"叙事模式"（Narrative Mode），而不是"小说的变体"。**

```
GenrePlugin
├── 定义"这个体裁的叙事单元是什么"
├── 定义"这个体裁需要哪些信息维度"
├── 提供"这个体裁的专属工具"
├── 提供"这个体裁的专属编辑器"
├── 提供"这个体裁的专属评审员"
└── 提供"这个体裁的专属知识图谱扩展"
```

*本文档是后续所有体裁开发的架构基础。*
