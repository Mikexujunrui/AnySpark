## 1. 欢迎使用火花

火花（AnySpark）是一个基于 AI 大模型的**全流程小说创作平台**。它不只是辅助写作的工具，更是一个能自主规划、执行和监督创作任务的智能引擎。

**火花能帮你做这些事：**

- 从灵感碎片到完整故事，一步步引导你构建世界观、角色和剧情
- 自动生成大纲、细纲，按章节写作，保持角色和世界观前后一致
- 8 位虚拟评审员从不同角度审查你的章节质量
- 开启 Autopilot 模式，让 AI 在后台自主完成批量写作
- 对已有内容进行精准修改，只改你想改的部分，不动其余原文
- 导入参考书，辅助同人创作和原著改写

**你需要准备什么：**

- 一台能上网的电脑（普通笔记本即可，不需要高端显卡）
- 一个 DeepSeek API Key（从 [platform.deepseek.com](https://platform.deepseek.com) 免费注册获取，新用户有免费额度）

## 2. 安装与部署

### 2.1 前置条件

| 你需要什么 | 说明 | 获取方式 |
|-----------|------|---------|
| **Docker Desktop** | 唯一必须安装的软件 | [Docker 官网](https://docs.docker.com/desktop/) 下载安装 |
| **DeepSeek API Key** | 用于调用 AI 大模型 | [platform.deepseek.com](https://platform.deepseek.com) 注册获取 |
| **网络连接** | 首次部署需下载约 1~2GB | 普通宽带即可 |

### 2.2 一键安装（推荐）

**Windows 用户：**

```powershell
# 右键 install.ps1 → "使用 PowerShell 运行"
# 或在命令行中执行：
powershell -ExecutionPolicy Bypass -File install.ps1
```

**Mac / Linux 用户：**

```bash
bash install.sh
```

脚本会自动完成所有步骤：检测 Docker → 引导你填入 API Key → 拉取镜像 → 启动服务。完成后在浏览器打开 `http://localhost:8190` 即可开始使用。

### 2.3 手动部署

如果你熟悉命令行，也可以手动操作：

```bash
# 1. 克隆仓库
git clone https://github.com/Mikexujunrui/AnySpark.git
cd AnySpark

# 2. 创建配置文件
cp .env.example .env
# 用记事本打开 .env，把 DEEPSEEK_API_KEY= 后面填上你的 API Key

# 3. 拉取镜像并启动
docker compose pull
docker compose up -d

# 4. 打开浏览器访问 http://localhost:8190
```

### 2.4 更新版本

```bash
docker compose pull        # 拉取最新镜像
docker compose up -d       # 重启服务
```

你的写作数据存储在 `data/` 目录中，更新不会丢失。

## 3. 快速上手：你的第一篇小说

下面带你走一遍从零到写出第一章的完整流程。

### 3.1 创建你的第一本书

1. 打开 `http://localhost:8190`
2. 点击左侧书架上的 **"+ 新建书籍"** 按钮
3. 输入书名（如"我的第一篇小说"），点击创建
4. 点击新创建的书，进入写作界面

### 3.2 告诉火花你的故事构想

在底部的对话框中，用自然语言告诉火花你的想法。例如：

> "我想写一个架空世界的故事。主角叫林风，是一个普通的大学生，某天突然发现自己能看到别人身上的'命运线'。他可以看到一个人即将遭遇的危险或机遇……"

火花会：
- 理解你的设定，自动提取角色、世界观信息
- 存入知识库，确保后续写作时不会忘记
- 可能会问你一些补充问题，帮助你完善设定

### 3.3 生成大纲

设定完善后，对火花说：

> "请为这个故事生成大纲，规划 30 章，分成 3 卷。"

火花会调用大纲生成工具，输出完整的章节大纲供你审阅。

### 3.4 写第一章

大纲确认后，说：

> "大纲很好！请开始写第一章。"

火花会：
- 从知识库中检索当前章节涉及的角色、地点、事件
- 在知识约束下生成第一章正文
- 写完后自动进行一致性校验

### 3.5 继续写下去

你可以让火花继续：

> "继续写第二章。"

也可以开启 Autopilot 让它自动批量写作（详见 [5.3 Autopilot 自主写作](#53-autopilot-自主写作)）。

## 4. 界面总览

火花采用**三面板分离式布局**，从左到右依次为：

### 左侧面板：导航区

- **书架**：管理你的所有书籍，切换写作项目
- **会话列表**：每本书可以有多个对话会话，方便按主题/阶段管理对话
- **素材面板**：管理上传的参考文件

### 中间面板：对话区（主工作区）

- **消息列表**：与 AI 的对话记录，支持富文本、剧情卡片、评审报告等嵌入展示
- **消息输入框**：底部输入你的指令，支持 `/` 斜杠命令快速调用功能
- **上下文栏**：显示当前写作的上下文信息（当前章节、活跃文风等）
- **进度指示器**：显示 AI 正在执行的操作

### 右侧面板：工具区

右侧包含多个标签页，按需切换：

| 标签页 | 功能 |
|--------|------|
| **章节** | 查看和管理所有章节，支持拖拽排序 |
| **大纲** | 查看三级大纲结构（卷→章→细纲） |
| **知识库** | 世界观设定、角色档案、地点、事件的 Wiki 式管理 |
| **可视化** | 时间线、角色关系图、角色热度图、世界地图、伏笔看板 |
| **评审** | 评审团面板，查看评审历史和报告 |
| **文风** | 管理文风模板和当前活跃文风 |
| **工作流** | 技能工作流管理 |
| **搜索** | 全文搜索章节内容 |
| **参考书** | 导入的参考书管理 |

## 5. 核心功能详解

### 5.1 对话式创作

火花的创作方式以**自然语言对话**为核心。你只需要像跟朋友聊天一样说出你的想法，火花就会理解并执行。

**对话中的富交互组件：**

- **剧情卡片**：火花规划剧情时，会生成 3~5 个不同走向的选项卡片，你可以点击选择
- **评审报告**：评审完成后，结果以卡片形式嵌入对话
- **章节 Diff**：修改章节后，显示变更对比卡片（绿色为新增，红色为删除）
- **任务进度**：Autopilot 执行时，显示进度条和当前步骤

**斜杠命令**：在输入框中输入 `/` 可以快速调用常用功能（如 `/write` 写作、`/review` 评审等）。

### 5.2 Agent 智能助手

火花的核心是一个**自主循环 Agent**。它不像普通 AI 聊天那样一问一答，而是：

1. 收到你的指令后，**自主规划**需要做什么（调用哪些工具、按什么顺序）
2. 每一步执行完，**自己判断**结果是否满意，是否需要再来一次
3. 所有步骤完成后，**主动汇报**完成情况

**安全保障机制：**

- **轮次上限**：写作任务最多 100 轮，其他任务最多 30 轮，防止无限循环
- **Token 预算**：累计消耗超过模型上下文 80% 时自动停止
- **死循环检测**：同一操作重复 3 次自动打断
- **幻觉防护**：检测 AI 是否伪造了不存在的工具调用
- **确定性终态**：每条任务路径都有明确的结束原因，不会"莫名停止"

**子 Agent 系统**：复杂任务可以拆分为子 Agent 并行处理，支持 5 种类型：

| 类型 | 用途 |
|------|------|
| general | 通用任务 |
| extract | 知识抽取 |
| plan | 剧情规划 |
| write | 写作 |
| research | 联网调研 |

### 5.3 Autopilot 自主写作

Autopilot 是火花的**后台自动写作引擎**。一句话指令，它就能在后台持久运行，自主完成多章写作。

**使用步骤：**

1. 在对话中输入你的写作指令，例如：

> "请按大纲写完全部 30 章，每章 2000~2500 字。"

2. 火花会生成一个**写作计划**，列出将要执行的步骤
3. 你确认计划后，火花开始在后台逐章写作
4. 你可以在右侧 **任务进度面板** 中查看实时进度
5. 每章完成后，内容自动保存，你可以随时审阅

**审核模式：**

| 模式 | 说明 |
|------|------|
| **hard** | 每步完成后暂停，等待你确认才继续 |
| **soft** | AI 自主推进，但遇到不确定的情况会问你 |
| **autonomous** | 全自动执行，无需人工干预 |

> ⚠️ **注意**：Autopilot 是实验功能，请在充分审核每步输出的前提下谨慎使用。人类作家的判断力始终是创作的最后一道闸门。

**自主模式开关**：在设置面板中可以开启/关闭"自主模式"（即允许 AI 无确认执行操作）。默认关闭，需要手动开启。

### 5.4 知识库与世界观

火花使用 **Neo4j 图数据库**存储你的世界观信息，确保角色关系、事件因果、时间线等信息始终一致。

**知识库包含：**

- **角色**：姓名、外貌、性格、能力、背景故事、与其他角色的关系
- **地点**：地理位置、描述、所属区域
- **物品**：重要道具、武器、功法等
- **组织**：势力、门派、家族等
- **事件**：关键历史事件、伏笔
- **概念**：世界观规则、魔法体系、社会制度等

**知识库的使用方式：**

- **自动提取**：你和 AI 对话中提到的角色、设定，火花会自动识别并存入知识库
- **手动编辑**：在右侧"知识库"面板中，可以查看、编辑、添加条目
- **写作约束**：写新章节时，火花会自动检索相关知识，确保写出的内容与设定一致
- **一致性校验**：写作完成后自动检测是否有与知识库矛盾的内容

**可视化面板**：在右侧面板中，你可以通过 Wiki 视图查看所有知识条目，支持编辑、删除和关联。

### 5.5 大纲与细纲

火花支持**三级大纲体系**：

| 层级 | 说明 |
|------|------|
| **卷（Volume）** | 全书的宏观结构，如"第一卷：崛起篇" |
| **章（Chapter）** | 每卷下的章节规划，包含章节标题和简介 |
| **细纲（Detailed Outline）** | 每章的详细剧情骨架，包括场景、情节节拍、关键对话 |

**使用方式：**

- 对火花说"请生成全书大纲"，AI 会自动规划分卷和章节
- 大纲生成后，可以继续要求"生成详细细纲"
- 在右侧"大纲"面板中，可以查看和编辑大纲结构
- 支持 AI 自动分卷：你可以指定每卷多少章，AI 自动分配

**剧情链（Plot Chain）**：每章写作完成后，火花会自动提取该章的情节节拍，形成结构化的剧情链，方便后续对比和审查。

### 5.6 剧情链精准编辑

传统 AI 改写最大的问题是：你让它改一段对话，它把整章都重写了。火花的剧情链编辑彻底解决了这个问题。

**工作流程：**

1. **拆解**：AI 将章节拆解为结构化节点（场景地点、出场人物、情节节拍、关键对话、情绪弧线），每个节点附带原文锚点
2. **标注**：对每个节点指定编辑策略——
   - `keep`：原样保留，不动
   - `tweak`：微调特定部分
   - `rewrite`：大幅改写
3. **改写**：AI 只改你标记的节点，其余原文纹丝不动
4. **审核**：每个节点的改动结果独立呈现，改错了只回滚那一个节点

**使用方式：**

> "请用剧情链编辑模式修改第三章，把主角和王老师的对话改得更温和一些，其他部分不动。"

**核心优势**：一条 3000 字的章节，可能只改其中 200 字的 3 个节点，而不是推倒重来。

### 5.7 评审团审查

火花内置了 **9 位虚拟评审员**，从不同角度审视你的章节质量。他们分为两大类：

**专业评审（4 位）：**

| 评审员 | 关注点 |
|--------|--------|
| 编剧 | 三幕结构、戏剧张力、场景转换 |
| 文学编辑 | 文笔质量、风格一致性、叙事视角 |
| 逻辑审校 | 情节漏洞、时间线矛盾、设定冲突 |
| 原著党 | 同人作品的人物还原度、世界观忠诚度 |

**读者评审（5 位）：**

| 评审员 | 关注点 |
|--------|--------|
| 爽文读者 | 爽感密度、节奏、打脸质量 |
| 情感型读者 | 情感共鸣、角色代入感 |
| 硬核党 | 世界观自洽、设定考据 |
| 挑刺王 | 原创性、角色智商、诚意度 |
| 休闲读者 | 可读性、吸引力、追更欲 |

**使用方式：**

> "请评审团审查第一章。"

火花会：
- 默认激活所有评审员并发评审
- 你也可以指定评审员，如"让编剧和爽文读者评审一下"
- 评审完成后，生成汇总报告（评分 + 共识 + 分歧 + 建议）
- 也可以在右侧"评审"面板中管理评审员（激活/停用/自定义）

**质量门控**：开启后，章节质量低于阈值时自动暂停 Autopilot，等待你处理。

### 5.8 章节版本控制

火花为每章提供 **Git 风格的版本控制**，每次修改自动创建新版本。

**功能：**

- **版本历史**：查看章节的所有历史版本，谁在什么时候改了什么
- **版本回退**：不满意当前版本，一键恢复到任意历史版本
- **版本对比**：对比两个版本之间的差异，新增/删除/修改一目了然
- **自动保存**：每次编辑自动创建新版本，不留痕迹地保护你的创作

**使用方式：**

在右侧"章节"面板中，点击任意章节，可以看到版本历史。点击"Diff"按钮对比差异，点击"回退"恢复到历史版本。

### 5.9 文风系统

火花预置了 **4 种文风模板**，你也可以创建自定义文风。

**系统预置文风：**

| 文风 | 特点 | 适用场景 |
|------|------|---------|
| **经典叙事** | 均衡节奏，因果清晰，三幕结构 | 日常、过渡、铺垫、通用 |
| **快节奏** | 短句、多动作、少描写 | 战斗、追逐、高潮、危机 |
| **黑暗风格** | 压抑氛围，强调代价与危险 | 悬疑、背叛、牺牲、灾难 |
| **文雅古典** | 长句、用典、意境渲染 | 回忆、独白、抒情、历史 |

**每本书可以独立设置活跃文风**。写作时，火花会自动根据当前文风调整叙事策略。

**使用方式：**

- 在右侧"文风"面板中查看和管理文风
- 对火花说"切换到快节奏文风"
- 文风支持 CRUD 操作：创建自定义文风、修改、删除
- 系统默认文风只读，自定义文风存储在 `data/styles/` 中

**叙事导演**：每个文风内置了 6 维度叙事策略控制（POV、节奏曲线、揭示密度、伏笔预算、章节弧线、语调指引），让 AI 严格遵循指定的叙事风格。

### 5.10 技能系统

技能是**预定义的复合工作流**，将多个工具调用组合成一个一键执行的操作。

**系统预装技能（7 个）：**

| 技能 | 用途 |
|------|------|
| `full_novel_import` | 导入完整小说：存档 + 知识提取 + 一致性校验 |
| `setting_extraction` | 纯设定提取：深度解析人物、世界观、事件 |
| `draft_polish` | 草稿精修：存为章节 + 提取新设定 + 建议续写方向 |
| `brainstorm_expand` | 灵感展开：记录灵感 + 关联角色 + 建议剧情方向 |
| `daily_writing` | 日常写作：续写 + 校验 + 保存 |
| `full_novel_reconstruct` | 拆书复写：拆解→提取→复写→对比 |
| `chapter_rewrite` | 单章复写：拆解→提取→复写→校验 |

**使用方式：**

- 火花会根据你的输入内容自动推荐合适的技能
- 你也可以直接说"用拆书复写技能处理这一章"
- 在右侧"工作流"面板中查看和管理技能

### 5.11 全书批量变换

火花提供 **5 个全书级变换工具**，让你用一句话修改整部作品。

| 工具 | 功能 | 示例 |
|------|------|------|
| `apply_directive_globally` | 一条指令应用到全书 | "把全书中所有'张三'改成'李四'" |
| `find_replace_book` | 全书查找替换 | 支持字面匹配和正则表达式 |
| `transform_chapters_batch` | 批量变换指定章节 | 支持 patch 精准编辑和 rewrite 全文重写 |
| `restyle_book` | 全书统一样式 | "把全书换成快节奏文风" |
| `summarize_book` | 全书摘要生成 | 生成长程上下文，辅助后续写作 |

**使用方式：**

> "把第 3 到第 15 章的对话风格统一为更冷峻的调子。"

火花会自动判断需要修改哪些章节，逐章应用修改，每章完成后生成 Diff 供你审查。

### 5.12 可视化工具

火花提供丰富的可视化面板，帮助你从不同维度理解自己的故事。

**时间线视图**：事件和章节在水平时间轴上的分布，标注伏笔与关键节点。支持缩放和拖拽，清晰展示创作脉络。

**角色关系图**：D3.js 力导向图，展示角色之间的互动关系网络。支持点击角色查看详情，拖拽节点调整布局。

**角色热度图**：角色在各章节的出场频次热力图，一眼看出哪些角色被冷落了，哪些角色出场过多。

**世界地图**：Leaflet 地理可视化 + 时间滑块，在 4D 维度（空间 + 时间）呈现你的世界观。适合构建复杂地理设定的奇幻/科幻作品。

**伏笔追踪看板**：伏笔→回收全链路追踪。记录每个伏笔的埋设位置、计划回收时间、实际回收状态，悬疑类创作的刚需工具。

**统计仪表盘**：字数趋势、章节分布、角色频次等多维度统计面板。

### 5.13 导入与导出

**导入：**

支持上传 `txt`、`docx`、`md` 格式的文件。火花会自动解析文档内容，提取知识并构建章节结构。

使用方式：点击左侧面板的"导入"按钮，或直接拖拽文件到对话框。

**导出：**

支持三种格式导出：
- **txt**：纯文本，适合进一步编辑
- **docx**：Word 文档，保留格式
- **markdown**：Markdown 格式，适合发布到网络平台

支持整本导出或单章导出。在章节面板中点击"导出"按钮即可。

### 5.14 参考书注入

如果你在进行**同人创作**或**原著改写**，参考书注入功能非常有用。

**工作流程：**

1. 上传参考书（原作文档）
2. 火花自动提取知识图谱（角色、事件、时间线）
3. 写作时，原著对应章节的完整内容注入 Prompt
4. AI 在理解原著角色语气和世界细节的前提下进行创作

**段落级对齐改写**：基于 diff-patch 技术，精准替换目标段落，不影响原文其他部分。

**使用方式**：在右侧"参考书"面板中管理参考书，导入后自动在你的写作中生效。

### 5.15 联网搜索

火花集成了联网搜索功能，帮助 AI 获取写作所需的真实资料。

**支持的功能：**

- **web_search**：搜索写作素材（如"17 世纪英国贵族礼仪"）
- **web_fetch**：抓取网页内容，提取纯文本
- **research 子 Agent**：进行多轮深度调研，汇总整理

**搜索后端**：Exa 和 Parallel 双后端，系统自动分配。

**使用方式**：在对话中直接让 AI 搜索，如"搜索一下宋朝的科举制度"，AI 会自动调用搜索工具。

> 联网搜索默认开启，可在 `.env` 中设置 `WEBSEARCH_ENABLED=false` 关闭。

### 5.16 剧情卡片推演

当你卡文或需要灵感时，火花可以生成多个剧情走向供你选择。

**交互流程：**

1. 对火花说"帮我规划一下接下来的剧情走向"
2. 火花生成 3~5 个不同方向的剧情卡片
3. 每张卡片包含：方向名称、描述、关键事件、基调标签、对后续的影响、创作风险
4. 你可以：点击选择某个方向 / 输入自定义方向 / 全部拒绝
5. 火花根据你的选择继续创作

**适用场景**：卡文破局、灵感验证、多方向探索、低成本试错。

### 5.17 草稿/定稿双轨

章节支持**草稿和定稿两种状态**，方便你管理创作流程。

- **草稿**：正在写作中，内容可能不完整
- **定稿**：已完成并通过评审，标记为稳定版本

**操作：**

- **升级（promote）**：草稿 → 定稿，锁定为稳定版本
- **降级（demote）**：定稿 → 草稿，重新开放编辑

前端状态栏以 Badge 标识区分两种状态，支持稳定版与草稿版并行管理。

### 5.18 叙事导演

每个文风内置了**6 维度叙事策略**，精确控制 AI 的写作风格：

| 维度 | 说明 | 可选值 |
|------|------|--------|
| **POV（视角）** | 叙事视角 | 第三人称有限 / 第三人称全知 / 第三人称电影化 |
| **Pacing（节奏）** | 节奏曲线 | 三幕结构 / 过山车式 / 慢热型 |
| **Reveal Density（揭示密度）** | 信息揭示密度 | 适中 / 密集 / 稀疏 |
| **Foreshadow Budget（伏笔预算）** | 每章伏笔数量 | 1~5 个 |
| **Chapter Arc（章节弧线）** | 章节结构 | 铺垫-发展-高潮-解决 等 |
| **Tone Guidance（语调指引）** | 语调建议 | 自然语言描述，如"保持理性克制" |

## 6. 配置说明

所有配置通过 `.env` 文件管理。部署后，你可以在项目根目录找到这个文件。

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DEEPSEEK_API_KEY` | **必填**，LLM API 密钥 | — |
| `DEEPSEEK_BASE_URL` | API 地址 | `https://api.deepseek.com` |
| `DEEPSEEK_MODEL` | Pro 模型（创作/规划） | `deepseek-v4-pro` |
| `DEEPSEEK_MODEL_FLASH` | Flash 模型（抽取/校验） | `deepseek-v4-flash` |
| `LLM_MODE` | 调度模式 | `split` |
| `NEO4J_PASSWORD` | Neo4j 数据库密码 | `novel_agent_2024!` |
| `SERVER_PORT` | 后端端口 | `8191` |
| `WEBSEARCH_ENABLED` | 启用联网搜索 | `true` |

**LLM 调度模式说明：**

| 模式 | Pro 模型负责 | Flash 模型负责 | 推荐场景 |
|------|-------------|---------------|---------|
| `split` | 创作、规划、评审 | 抽取、校验、分类 | **推荐**，质量与成本平衡 |
| `pro` | 全部任务 | — | 追求最高质量 |
| `flash` | — | 全部任务 | 快速迭代、低成本试错 |
| `custom` | 自由组合 | 自由组合 | 接入其他 OpenAI 兼容 API |

**端口说明：**

| 服务 | 端口 |
|------|------|
| 前端界面 | 8190 |
| 后端 API | 8191 |
| Neo4j Browser | 7474 |
| Neo4j Bolt | 7687 |

## 7. 常见问题

### 7.1 为什么需要 Docker？

火花依赖 Neo4j 图数据库来管理角色关系、世界观和时间线。Neo4j 需要 Java 运行时，Docker 把它打包好了，你只需安装 Docker 一个软件。如果不用 Docker，你需要手动安装 Java + Neo4j + Python 3.11+ + Node.js 20+，比装 Docker 复杂得多。

### 7.2 需要 GPU 吗？

不需要。火花调用云端 DeepSeek API 进行 AI 推理，本地只运行前端界面和后端任务调度，普通 CPU 笔记本即可流畅运行。

### 7.3 端口被占用怎么办？

默认端口：前端 8190、后端 8191、Neo4j 7474/7687。如果冲突，编辑 `.env` 修改 `SERVER_PORT`，编辑 `docker-compose.yml` 修改端口映射。

### 7.4 如何备份数据？

```bash
# 备份整个 data 目录即可
cp -r data/ data_backup_$(date +%Y%m%d)/
cp .env .env.backup
```

### 7.5 启动失败怎么办？

```bash
# 查看日志定位问题
docker compose logs

# 常见原因：
# 1. DEEPSEEK_API_KEY 未填写或无效 → 检查 .env 文件
# 2. 端口冲突 → 关闭占用端口的程序，或修改端口映射
# 3. 磁盘空间不足 → 清理 Docker 缓存：docker system prune -a
```

### 7.6 我的数据安全吗？

火花采用**代码与数据严格分离**的设计。`data/` 目录已被 `.gitignore` 完全排除，不会上传到 Git。你的小说、角色、设定完全归你所有，存储在你自己的电脑上。

### 7.7 可以更换 AI 模型吗？

可以。火花支持任何 OpenAI 兼容的 API。将 `LLM_MODE` 设为 `custom`，然后修改 `DEEPSEEK_BASE_URL`、`DEEPSEEK_MODEL`、`DEEPSEEK_MODEL_FLASH` 指向你的 API 提供商即可。

### 7.8 墙内 Docker 拉取镜像慢怎么办？

打开 Docker Desktop 设置 → Docker Engine → 添加国内镜像源：

```json
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://mirror.ccs.tencentyun.com"
  ]
}
```

### 7.9 如何控制 AI 写作成本？

- 使用 `split` 模式：创作用 Pro 模型，简单的抽取/校验用 Flash 模型（更便宜）
- 使用 `flash` 模式：全部用 Flash 模型，成本最低
- 合理设置轮次上限和 Token 预算
- 短章节测试后再写长章节
